# 《时间泡泡（Time Bubble）》产品设计文档

> Version 1.0
>
> **一句话介绍：**
>
> **让时间像泡泡一样流动。**

---

# 一、产品定位

《时间泡泡》不是 Todo，不是番茄钟，也不是传统 Time Tracker。

它是一款用于记录：

> **此刻，我的时间属于谁。**

的时间可视化应用。

传统产品关注：

> 我要完成什么？

《时间泡泡》关注：

> 我的时间，现在流向哪里？

整个产品围绕 **时间流（Time Flow）** 展开，而不是围绕任务（Task）展开。

---

# 二、产品理念

一天，就是一颗泡泡。

时间不断流逝。

泡泡不断成长。

用户唯一需要做的，就是告诉它：

> 此刻，这段时间属于谁。

例如：

```
学习

娱乐

工作

阅读

运动
```

除此之外，不需要：

- 点击开始
- 点击结束
- 创建任务
- 完成任务

时间一直都在流动。

用户只负责改变时间流向。

---

# 三、核心概念

整个产品只有三个核心对象。

## 1、Bubble Type（时间类型）

Bubble Type 表示：

**时间属于哪个分类。**

例如：

```
学习

娱乐

工作

阅读

运动
```

用户可以自由创建。

每个 Bubble Type 包含：

- 名称
- 图标
- 颜色

例如：

```
📖 学习

颜色：

#5B8DEF
```

注意：

Bubble Type 不记录时间。

它只是一个分类。

---

## 2、Time Segment（时间片段）

这是整个产品最核心的数据。

Time Segment 表示：

**一段连续属于同一个 Bubble Type 的时间。**

例如：

```
08:00

↓

学习

↓

09:30
```

产生：

```
Segment

学习

08:00

~

09:30
```

随后：

```
09:30

↓

娱乐
```

学习 Segment 结束。

娱乐 Segment 开始。

一天实际上就是：

```
学习

↓

娱乐

↓

学习

↓

工作

↓

娱乐

↓

学习
```

很多 Segment 连在一起。

而不是：

```
学习：

5小时

娱乐：

2小时
```

统计页面再根据 Segment 计算：

```
学习：

5小时
```

因此：

整个产品记录的是：

> 时间片段（Segment）

不是：

> 时间累计。

---

## 3、Daily Bubble（每日泡泡）

每天只有一个：

Daily Bubble。

它代表：

今天。

泡泡随着时间不断成长。

最终：

爆炸。

结束今天。

第二天：

重新生成新的泡泡。

# 四、产品原则

整个产品遵循以下原则。

---

## 原则一

任何时间都必须属于某个 Bubble。

不存在：

```
暂停
```

不存在：

```
没有记录
```

默认始终存在：

```
无所事事
```

它默认隐藏。

只要用户没有主动切换。

时间就属于：

无所事事。

因此：

每天时间一定完整。

---

## 原则二

首页只展示：

> 当前。

首页不展示：

```
今天学习：

5小时
```

首页只展示：

```
学习

00:26:31
```

这里表示：

当前这一次学习。

已经持续：

26分钟31秒。

切换 Bubble Type 后：

重新开始计时。

例如：

```
学习

00:26:31

↓

点击娱乐

↓

娱乐

00:00:00
```

---

## 原则三

统计页面才展示：

今天。

例如：

```
学习：

4小时

娱乐：

2小时

工作：

5小时
```

首页永远关注：

```
现在。
```

统计页面关注：

```
今天。
```

---

## 原则四

外环记录：

时间流。

不是：

时间统计。

例如：

```

无所事事

↓

08:20

学习

↓

09:10

娱乐

↓

09:40

学习

↓

11:20

工作
```

外环应该记录：

```
灰

↓

蓝

↓

紫

↓

蓝

↓

绿
```

注意：

学习出现两次。

这是正常的。

因为：

外环记录的是：

Time Segment。

不是：

Bubble Type。

---

## 原则五

所有统计都由 Segment 计算。

例如：

今天：

```

08:20~09:10

50分钟

-------

09:40~11:20

100分钟
```

统计：

```

150分钟
```

首页：

仍然不会显示：

```
150分钟
```

只有：

日统计。

周报。

月报。

年报。

才展示。

---

## 原则六

动画必须服务于：

时间流动。

不要为了动画而动画。

例如：

Bubble 融合。

表示：

时间流向改变。

Bubble 爆炸。

表示：

今天结束。

颜色扩散。

表示：

时间正在流动。

动画都具有明确含义。

# 十、大泡泡外环（Bubble Ring）

这是整个产品最重要的部分。

外环不是：

分类统计。

> **今天时间流动的轨迹。**

Time Segment。

不是：

Bubble Type。

---

## 外环增长规则

时间不断流逝。

外环不断增长。

例如：

今天：

16:00

开始。

结束：

22:00。

总长度：

```
6小时
```

当前：

17:30。

那么：

已经过去：

```

```

外环应该完成：

```

```

而不是：

24小时中的：

6%。

外环长度永远根据：

今天 Bubble 生命周期。

自动计算。

# 五十五、总结

《时间泡泡》的核心不是：

> 时间统计。

而是：

> 时间流动。

整个产品应该让用户感受到：

时间一直在缓慢流动。

Bubble 一直在成长。

颜色不断融合。

直到：

今天结束。

Bubble 爆炸。

最终：

变成今天的一颗时间泡泡。

用户记录的不是：

Todo。

不是：

番茄钟。

不是：

任务。

而是：

> **今天，我的时间最终流向了哪里。**

因此：

整个产品必须始终围绕：

> **Time Flow（时间流）**

而不是：

> **Time Statistics（时间统计）**

设计。
