# 时间泡泡 Time Bubble 项目开发指导文档

> 本文档是后续开发项目的主参考文档。  
> 目标不是继续扩写产品概念，而是把产品理解转化为可执行的开发边界、页面结构、数据模型、交互规则和验收标准。

---

## 1. 项目定位

时间泡泡是一款“时间流向记录”应用。

它要回答的问题不是：

```text
我今天完成了什么？
```

而是：

```text
此刻，我的时间属于谁？
```

产品核心是 **Time Flow 时间流**，不是 Todo，不是番茄钟，不是普通时间统计器。  
用户不创建任务，不点击开始/结束，不做完成勾选，只在不同时间类型之间切换。

---

## 2. 开发时必须守住的核心原则

### 2.1 首页只表达“现在”

首页只能显示当前 Segment 的状态：

```text
当前类型
当前这一段已持续多久
```

首页不能显示：

- 今天学习累计多久
- 各类型占比
- 排行榜
- 统计卡片
- 任务列表

首页的主问题永远是：

```text
现在，我的时间属于谁？
```

---

### 2.2 统计只来自 Segment

数据库不直接保存：

```text
学习：5小时
```

数据库只保存一段段 Time Segment：

```text
08:00 - 09:00 学习
10:00 - 12:00 学习
```

统计页需要展示“学习共 3 小时”时，实时从 Segment 聚合计算。

---

### 2.3 时间不能暂停

产品里没有暂停态。  
任何时间都必须归属于某个 Bubble Type。

默认内置一个隐藏类型：

```text
无所事事
```

用户未选择其他类型时，当前时间自动归入“无所事事”。

---

### 2.4 外环不是饼图

首页大泡泡外环表示“时间流轨迹”，不是分类统计。

例如当天 Segment 是：

```text
无所事事 -> 学习 -> 娱乐 -> 学习 -> 工作
```

外环必须表现为：

```text
灰 -> 蓝 -> 紫 -> 蓝 -> 绿
```

即使“学习”出现两次，也不能合并成一段蓝色。

---

### 2.5 泡泡不是按钮

Bubble 是产品的核心生命体。  
它可以点击，但视觉上不能像普通按钮。

Bubble 应该具备：

- 漂浮感
- 呼吸感
- 柔和光晕
- 半透明玻璃质感
- 被切换时的融合动画

---

## 3. MVP 第一版范围

第一版目标：做出可真实使用的本地版时间泡泡。

### 必须完成

- 首页 Home
- 大泡泡 Main Bubble

- 时间类型 Bubble Type 创建

- 切换 Bubble Type
- Segment 本地持久化

- 云同步
- AI 总结
- 社交分享
- 成就系统
- 主题市场

## 5. 数据模型

### 5.1 BubbleType

```ts
interface BubbleType {
  id: string;
  name: string;
  color: string;
  icon: string;
  hidden: boolean;
  createdAt: number;
  updatedAt: number;
}
```

说明：

- 表示一种时间类型。
- `hidden` 用于隐藏“无所事事”等系统类型。
- `icon` 第一版可以用 emoji 字符串，也可以后续换成图标 key。

---

### 5.2 TimeSegment

```ts
interface TimeSegment {
  id: string;
  bubbleTypeId: string;
  startTime: number;
  endTime?: number;
  dayId: string;
  createdAt: number;
  updatedAt: number;
}
```

说明：

- 核心数据对象。
- 当前正在进行的 Segment 没有 `endTime`。
- 用户切换 Bubble Type 时，关闭旧 Segment，再创建新 Segment。
- `dayId` 用于快速归档到某一天，例如 `2026-07-10`。

---

### 5.3 DailyBubble

```ts
interface DailyBubble {
  id: string;
  dayId: string;
  startTime: number;
  endTime: number;
  exploded: boolean;
  createdAt: number;
  updatedAt: number;
}
```

说明：

- 表示某一天的泡泡生命周期。
- `startTime` 是当天用户第一次打开应用并生成泡泡的时间。
- `endTime` 来自设置里的每日结束时间。
- 首页外环进度基于 `startTime` 到 `endTime`，不是固定 24 小时。

---

### 5.4 Settings

```ts
interface Settings {
  id: string;
  bubbleEndTime: string;
  themeMode: "light" | "dark" | "system";
  animationSpeed: "fast" | "normal" | "slow";
  soundEnabled: boolean;
  ambientEnabled: boolean;
  floatingEnabled: boolean;
  updatedAt: number;
}
```

默认值建议：

```ts
const defaultSettings = {
  bubbleEndTime: "22:00",
  themeMode: "dark",
  animationSpeed: "normal",
  soundEnabled: true,
  ambientEnabled: false,
  floatingEnabled: true,
};
```

## 6. 数据库设计

IndexedDB 表：

```text
bubbleTypes
segments
dailyBubbles
settings
```

不要在第一版建立统计表。

统计由以下数据实时计算：

- 指定日期内的 segments
- bubbleTypes 颜色和名称
- dailyBubble 生命周期

后续如果数据量变大，再增加 cache 表。

## 7. 核心业务流程

### 7.1 应用首次启动

1. 初始化默认 Settings。
2. 创建隐藏 Bubble Type：无所事事。
3. 创建今天的 DailyBubble。
4. 创建当前 Segment，类型为“无所事事”。
5. 首页显示 Main Bubble 和添加按钮。

---

### 7.2 用户创建 Bubble Type

1. 点击添加 Bubble。
2. 输入名称、选择颜色、选择图标。
3. 保存 BubbleType。
4. 新 Bubble 出现在首页外围。
5. 不自动切换，除非用户继续点击该 Bubble。

---

### 7.3 用户切换 Bubble Type

1. 读取当前未结束 Segment。
2. 将当前时间写入旧 Segment 的 `endTime`。
3. 创建新的 Segment。
4. 更新当前 Bubble Type。
5. Main Bubble 计时归零。
6. Bubble Ring 新增一段颜色。
7. 播放融合动画。

注意：如果用户点击的就是当前类型，第一版可以不做任何操作，避免生成重复 Segment。

---

### 7.4 到达每日结束时间

1. 结束当前 Segment。
2. 将 DailyBubble 标记为 `exploded: true`。
3. 播放 Bubble Explosion。
4. 展示今日总结。
5. 当用户第二天再次进入时，创建新的 DailyBubble 和新的默认 Segment。

第一版可以先做自动结束和总结页，Explosion 动画可在后续阶段增强，但不能删除这个产品仪式。

## 8. 页面设计与功能边界

### 8.1 Home 首页

首页功能：

- 展示当前 Main Bubble
- 展示当前 Segment 已持续时间
- 展示 Bubble Ring 时间外环
- 展示 Orbit Bubble
- 支持切换 Bubble Type
- 支持添加 Bubble Type
- 进入 Calendar
- 进入 Settings

首页不能出现：

- 今日累计统计
- 复杂图表
- 列表型任务
- 底部导航
- 大量说明文字

验收标准：

- 用户打开首页后，3 秒内能理解当前时间属于哪个类型。
- 切换类型后，中央计时归零。
- 外环按 Segment 顺序增加颜色段。
- 页面整体感觉是“漂浮的泡泡世界”，不是表单界面。

---

### 8.2 Calendar 日历

日历功能：

- 按月份展示每天的历史 Bubble。
- 每天用一个小泡泡表示。
- 泡泡颜色取当天占比最高的 Bubble Type。
- 今天的泡泡有特殊光环。
- 点击某一天进入 Daily Detail。

验收标准：

- 日历不应该像传统数字格子表。
- 用户能通过颜色大致看出过去时间流向。
- 空记录日期使用低饱和灰色。

---

### 8.3 Daily Detail 当天详情

详情功能：

- 展示日期、星期。
- 展示总记录时长。
- 展示各 Bubble Type 累计时长和占比。
- 展示 Timeline。
- 支持轻量编辑 Segment。

Timeline 是详情页核心，必须按 Segment 顺序展示。

验收标准：

- 能清楚看到一天时间是如何流动的。
- 同一类型多次出现时，Timeline 保持多段展示。
- 统计结果由 Segment 聚合得出。

---

### 8.4 Settings 设置

设置功能：

- 修改每日结束时间。
- 修改主题模式。
- 修改动画速度。
- 开关音效。
- 开关背景氛围音。
- 开关漂浮动画。

验收标准：

- 设置页简洁，不做成复杂偏好中心。
- 每日结束时间是视觉和交互上的重点。

---

## 9. 视觉实现方向

整体风格：

```text
深色梦幻背景
玻璃拟态
液态泡泡
柔和光晕
少文字
少卡片
低边界感
```

避免：

- 管理后台风格
- 强边框卡片堆叠
- 表格化布局
- 传统 Todo 风
- 纯黑背景
- 过度紫蓝单一配色

颜色建议：

```text
背景深蓝黑：#0F172A
背景辅助：#1E293B
学习：#5B8DEF
工作：#4CAF50
娱乐：#A855F7
阅读：#F59E0B
运动：#EF4444
无所事事：#94A3B8
```

字体：

```text
正文：Inter / HarmonyOS Sans / PingFang SC
时间数字：JetBrains Mono 或其他等宽字体
```

## 10. 动画实现优先级

### 第一优先级

- Main Bubble 呼吸
- Orbit Bubble 漂浮
- 切换 Bubble 时的融合感
- 外环随时间增长

### 第二优先级

- 页面从 Bubble 展开
- 背景粒子缓慢移动
- Bubble Ring 液态边缘

### 第三优先级

- Bubble Explosion
- 音效
- 今日结束总结动画

动画原则：

- 不要突然消失。
- 不要简单 display none。
- 不要快速闪烁。
- 所有变化尽量柔和、缓慢、有语义。

## 13. 时间与日期规则

### dayId

统一使用本地日期：

```text
YYYY-MM-DD
```

### DailyBubble 生命周期

假设每日结束时间是 22:00。

如果用户 08:00 打开：

```text
08:00 - 22:00
```

如果用户 16:00 第一次打开：

```text
16:00 - 22:00
```

外环 100% 对应这个生命周期，而不是 24 小时。

### 跨过结束时间

如果当前时间已经超过今日结束时间，但今天尚未结束：

- 应立即触发结束流程。
- 不继续让当前 Segment 无限增长。

---

## 14. 统计计算规则

单个 Segment 时长：

```ts
duration = (segment.endTime ?? now) - segment.startTime;
```

某个 Bubble Type 的累计时长：

```text
筛选同一 dayId 下该 bubbleTypeId 的 segments
逐段计算 duration
求和
```

日历泡泡主色：

```text
取当天累计时长最高的 Bubble Type 颜色
```

Timeline：

```text
按 startTime 升序排列 segments
不合并相同类型
```

## 15. 第一版验收清单

### 功能验收

- 首次启动能自动创建默认数据。
- 用户能创建时间类型。
- 用户能切换当前时间类型。
- 切换后旧 Segment 有 endTime，新 Segment 开始。
- 当前 Segment 计时实时更新。
- 外环按 Segment 顺序显示颜色段。
- 日历能展示历史日期泡泡。
- 详情页能展示 Timeline 和统计。
- 设置页能修改每日结束时间。

### 产品原则验收

- 首页不展示累计统计。
- 外环不是饼图。
- 相同类型的多个 Segment 不被合并。
- 时间不存在暂停状态。
- 无所事事作为默认隐藏类型存在。

### 视觉验收

- 首页不是 Todo 风格。
- 页面没有底部导航。
- Main Bubble 有明显中心地位。
- Orbit Bubble 有漂浮感。
- 整体风格偏梦幻、柔和、玻璃质感。

## 17. 后续开发时的判断标准

当实现细节不确定时，优先问这三个问题：

1. 这个设计是否强化“时间正在流动”？
2. 这个信息是否应该出现在首页，还是应该放到详情页？
3. 这个功能会不会把产品拉回 Todo、番茄钟或传统统计工具？

如果答案会削弱时间流体验，就先不做。
