# 时间泡泡 Time Bubble 产品设计整理版

> 用途：给产品、开发、UI 设计师或画图 AI 快速理解产品方向。  
> 核心目标：把原始设计文档中分散的概念、页面、交互、视觉风格整理成一份更可读、更适合出 APP 效果图的说明。

---

## 1. 产品一句话

**时间泡泡是一款记录“此刻我的时间流向哪里”的时间可视化应用。**

它不是 Todo，不是番茄钟，也不是传统时间统计工具。  
它的核心不是“我完成了什么任务”，而是：

> **现在，我的时间属于谁？**

用户不需要创建任务、不需要点击开始/结束、不需要打卡。  
时间始终在流动，用户只需要在不同的时间类型之间切换。

---

## 2. 产品核心隐喻

**一天就是一颗泡泡。**

从用户打开应用开始，今天的泡泡开始成长。  
时间不断流动，泡泡不断变大。  
用户每次切换时间类型，泡泡的颜色和轨迹都会发生变化。

到达用户设定的每日结束时间后，今天的泡泡会完成一次柔和的爆裂仪式，变成日历里的一颗历史泡泡。

第二天，新的泡泡重新生成。

---

## 3. 核心体验关键词

- **现在感**：首页永远只关注当前这一段时间。
- **流动感**：时间像液体一样持续流动。
- **生命感**：泡泡会呼吸、漂浮、融合、成长。
- **仪式感**：一天结束时，泡泡爆裂并生成今日总结。
- **低负担**：用户只需要切换时间类型，不需要管理任务。
- **情绪价值**：它更像一个陪伴式时间世界，而不是效率工具。

---

## 4. 目标用户

适合：

- 学生
- 自由职业者
- 开发者
- 创作者
- 想观察自己时间流向的人
- 喜欢记录生活节奏的人

不适合：

- 企业项目管理
- OKR 管理
- 团队协作
- 复杂任务拆解

---

## 5. 三个核心数据对象

### 5.1 Bubble Type 时间类型

表示“时间属于哪个分类”。

示例：

- 学习
- 娱乐
- 工作
- 阅读
- 运动
- 无所事事

每个时间类型包含：

- 名称
- 图标
- 颜色

注意：Bubble Type 只是分类，本身不记录时间。

---

### 5.2 Time Segment 时间片段

这是产品最核心的数据。

Time Segment 表示：

> 一段连续属于同一个 Bubble Type 的时间。

示例：

```text
08:00 - 09:30  学习
09:30 - 10:10  娱乐
10:10 - 11:40  学习
```

即使“学习”出现两次，也应该是两个不同的 Segment。  
不要自动合并，因为它们发生在不同时间段。

所有统计都从 Segment 计算，而不是直接保存“学习共 5 小时”。

---

### 5.3 Daily Bubble 每日泡泡

每天只有一颗 Daily Bubble。  
它代表今天。

它从用户第一次打开应用时开始生成，到用户设定的每日结束时间结束。  
结束后，泡泡爆裂，进入今日总结，并成为日历中的一个历史泡泡。

---

## 6. 关键产品原则

### 原则 1：首页只展示“当前”

首页只显示当前这一段 Segment 的信息。

正确：

```text
📖 学习
00:26:31
```

含义：当前这一次学习已经持续 26 分钟 31 秒。

错误：

```text
今天学习 5 小时
```

累计时间只能出现在统计页、今日详情、周报、月报、年报中。

---

### 原则 2：任何时间都必须属于某个 Bubble

不存在“暂停”。  
不存在“没有记录”。

默认始终存在一个隐藏类型：

```text
无所事事
```

当用户没有主动选择时间类型时，时间默认归入“无所事事”。

---

### 原则 3：外环不是饼图，而是时间流轨迹

首页大泡泡外面有一圈时间环。  
它记录今天时间流动的顺序，而不是分类占比。

示例：

```text
无所事事 -> 学习 -> 娱乐 -> 学习 -> 工作
```

外环应显示为：

```text
灰 -> 蓝 -> 紫 -> 蓝 -> 绿
```

这里“学习”出现两次是正确的，因为这是两个不同的时间片段。

---

### 原则 4：动画必须有意义

动画不是装饰，而是表达产品语义。

- 泡泡漂浮：时间仍在流动
- 泡泡呼吸：今天仍在成长
- 泡泡融合：时间流向发生改变
- 颜色扩散：当前时间类型正在占据这一刻
- 泡泡爆裂：今天结束

---

## 7. 主要页面结构

产品页面建议保持克制，第一版只需要 4 个核心页面：

1. 首页 Home
2. 日历 Calendar
3. 今日/某日详情 Daily Detail
4. 设置 Settings

首页承担 80% 的使用时间。  
其他页面主要负责回顾、统计和配置。

---

## 8. 首页 Home

首页是产品最重要的页面。  
它应该像一个缓慢流动的泡泡世界，而不是工具软件面板。

### 8.1 首页目标

让用户一打开就知道：

> 此刻，我的时间属于谁。

首页不要出现：

- 任务列表
- 统计卡片
- 今日累计时间
- 底部 Tab
- 复杂按钮
- 大量文字说明

---

### 8.2 手机端布局

```text
┌────────────────────────────┐
│        📅          ⚙️       │
│                            │
│                            │
│         ◎ 大泡泡 ◎          │
│      📖 学习                │
│      00:26:31              │
│                            │
│   📖          🎮            │
│                            │
│         ＋                  │
│                            │
│   💼          🏃            │
└────────────────────────────┘
```

组成：

- 顶部：日历入口、设置入口
- 中央：Main Bubble
- 中央外环：Bubble Ring
- 周围：Orbit Bubble
- 背景：深色渐变、光晕、粒子、透明泡泡

---

### 8.3 Main Bubble 大泡泡

Main Bubble 表示今天，也表示当前正在流动的时间。

中央内容：

```text
📖
学习
00:26:31
```

其中：

- 图标：当前时间类型图标
- 名称：当前时间类型名称
- 时间：当前 Segment 已持续时长

切换到另一个时间类型后，中央计时归零。  
如果稍后再次切回“学习”，也重新从 00:00 开始，而不是继续累计。

---

### 8.4 Bubble Ring 时间外环

外环表示今天的时间流轨迹。

它随着时间逐渐增长。  
增长比例基于今天 Bubble 的生命周期，而不是固定 24 小时。

示例：

- 用户 16:00 第一次打开
- 每日结束时间 22:00
- 今天 Bubble 生命周期为 16:00 - 22:00，共 6 小时
- 17:30 时已经过去 1.5 小时
- 外环进度为 25%

外环颜色按 Segment 顺序排列：

```text
灰色 无所事事
蓝色 学习
紫色 娱乐
蓝色 学习
绿色 工作
```

外环视觉应该像柔和液态轨迹，不要像普通进度条或饼图。

---

### 8.5 Orbit Bubble 小泡泡

小泡泡围绕大泡泡漂浮，代表可切换的 Bubble Type。

每个小泡泡显示：

```text
📖
学习
```

不要显示累计时长，不要显示百分比。

首页建议最多展示 8 个小泡泡。  
更多分类可进入“更多”管理，避免页面拥挤。

---

### 8.6 添加 Bubble

第一次进入应用时，外围只有一个“+”泡泡。  
用户点击后创建时间类型。

创建内容：

- 名称
- 图标
- 颜色

创建成功后，新的小泡泡立刻出现在主泡泡周围。

---

## 9. Bubble 切换交互

用户点击某个小泡泡，例如“娱乐”。

系统执行：

1. 结束当前 Segment，记录 endTime
2. 创建新的 Segment，类型为“娱乐”
3. “娱乐”小泡泡飞向中央
4. 小泡泡与 Main Bubble 融合
5. Main Bubble 颜色扩散为娱乐色
6. 中央图标和文字切换
7. 中央计时归零
8. 旧的 Bubble 从中央退出，回到外围

建议动画时长：500 - 800ms。  
不要瞬间切换，不要简单淡入淡出。  
它应该像两个泡泡融合，而不是按钮状态变化。

---

## 10. 日历 Calendar

日历不是传统数字日历，而是一组历史泡泡。

每一天是一颗小泡泡：

```text
○ ○ ● ○ ◎
● ○ ○ ○ ○
○ ◎ ○ ● ○
```

颜色代表当天占比最高的时间类型：

- 蓝色：学习最多
- 紫色：娱乐最多
- 绿色：工作最多
- 橙色：阅读最多
- 灰色：记录较少或无记录

状态：

- 普通日：普通泡泡
- 今天：带光环的泡泡
- 未来：弱化的小点
- 无记录：灰色泡泡

点击某天泡泡后，泡泡放大并进入当天详情。

---

## 11. Daily Detail 当天详情

详情页展示首页不展示的统计信息。

内容包括：

- 日期和星期
- 总记录时长
- 当天主导时间类型
- 时间类型占比环形图
- 完整时间流 Timeline
- 每个 Bubble Type 的累计时长

其中 Timeline 最重要，因为它展示真实的时间流顺序：

```text
08:00  无所事事
08:20  学习
09:10  娱乐
09:40  学习
11:20  工作
```

支持对 Segment 做轻量编辑：

- 修改分类
- 修改开始/结束时间
- 删除误触记录

---

## 12. Settings 设置

设置页保持极简。  
最重要的设置是：

```text
每日结束时间 Bubble End Time
```

例如：

```text
22:00
```

含义：每天 22:00，今天的 Bubble 爆裂并结束。

其他设置建议只保留：

- 动画速度：快 / 正常 / 慢
- 深色模式：浅色 / 深色 / 自动
- 背景音乐：开 / 关
- 音效：开 / 关
- 漂浮动画：开 / 关

不要放大量高级设置。

---

## 13. 每日结束 Bubble Explosion

每日结束是产品的仪式感核心。

到达用户设置的结束时间后，自动触发 Bubble Explosion。

动画过程：

1. Main Bubble 停止呼吸
2. 泡泡逐渐变亮
3. 泡泡缓慢膨胀
4. 表面出现细微裂纹
5. 泡泡柔和爆裂
6. 小泡泡碎片飞向屏幕四周
7. 背景慢慢变暗
8. 出现月亮或夜晚氛围
9. 进入今日总结

爆裂不是剧烈爆炸，而是肥皂泡破裂般的柔和仪式。

今日总结文案方向：

```text
今天结束啦

你的时间最终变成了这样一颗 Bubble
```

## 14. 视觉设计语言

整体方向：

> Glassmorphism 玻璃拟态 + Dream UI 梦幻风格 + Liquid Glass 液态玻璃感

视觉参考气质：

- Apple Vision Pro
- iOS Liquid Glass
- 柔和发光的深色空间
- 半透明玻璃界面
- 轻盈、安静、漂浮

不要像：

- 后台管理系统
- Todo List
- Ant Design
- Material Design
- 传统打卡软件
- 数据看板首页

---

## 15. 颜色规范

### 背景

建议深色渐变，不要纯黑。

```text
#0F172A
#1E293B
#111827
```

背景可加入：

- 模糊光晕
- 微弱粒子
- 半透明泡泡
- 柔和渐变

### Bubble 默认颜色

```text
学习：#5B8DEF  蓝色
工作：#4CAF50  绿色
娱乐：#A855F7  紫色
阅读：#F59E0B  橙色
运动：#EF4444  红色
无所事事：#94A3B8  灰色
```

当前 Bubble Type 会轻微影响页面整体氛围色。  
例如当前是学习，背景略偏蓝；当前是娱乐，背景略偏紫。  
颜色变化应使用柔和过渡，避免突变。

---

## 16. 字体与 UI 细节

字体建议：

- Inter
- HarmonyOS Sans
- PingFang SC

数字时间建议使用等宽字体：

- JetBrains Mono

圆角：

```text
Bubble：9999px
Dialog：24px
Button：16px
```

阴影和光晕：

- 使用柔和发光
- 避免重黑阴影
- 可使用当前 Bubble 颜色的透明发光

毛玻璃：

- Dialog 和 Panel 使用 backdrop-filter blur
- 保持半透明
- 不要使用厚重纯色块

---

## 17. 动画规范

Main Bubble：

- 始终轻微呼吸
- 周期 4 - 6 秒
- Scale 在 99% - 101% 之间
- 不要明显跳动

Orbit Bubble：

- 始终缓慢漂浮
- X 方向约 ±6px
- Y 方向约 ±8px
- Rotation 约 ±2°
- 周期 8 - 15 秒
- 不同泡泡速度不同，避免同步

切换 Bubble：

- 总时长约 600ms
- 小泡泡放大、飞向中央、融合
- Main Bubble 颜色扩散
- 文字和计时器切换

页面切换：

- 不要普通 Fade
- 建议从某个 Bubble 放大展开
- 返回时收缩回 Bubble

Explosion：

- 总时长约 3 秒
- 柔和、慢速、有仪式感

## 21. 给画图 AI 的理解重点

画图 AI 需要抓住以下核心：

1. 这不是 Todo App，而是一个“时间流动可视化 App”。
2. 首页中心是一颗巨大的玻璃泡泡，代表今天。
3. 大泡泡中央显示当前时间类型、图标和当前持续时间。
4. 大泡泡外环是今天的时间流轨迹，不是饼图。
5. 周围漂浮多个小泡泡，代表可切换的时间类型。
6. 整体风格是深色梦幻、液态玻璃、柔和光晕、漂浮粒子。
7. UI 要少文字、少按钮、少卡片，重点是沉浸感和生命感。
8. 统计信息不要放首页，要放到详情页。
9. 日历页中，每一天是一颗小泡泡。
10. 今日结束时，有一个柔和的泡泡爆裂仪式。

## 22. 建议让画图 AI 输出的 5 张效果图

### 图 1：首页默认状态

手机 APP 界面，深色梦幻背景，中央一颗巨大的半透明玻璃泡泡，泡泡内部显示“📖 学习 00:26:31”，大泡泡外侧有彩色液态时间环，周围漂浮学习、娱乐、工作、运动等小泡泡，顶部只有日历图标和设置图标。整体风格像 Apple Vision Pro + iOS Liquid Glass，柔和发光，少文字，无底部导航。

展示用户从“学习”切换到“娱乐”的动态中间状态。紫色娱乐小泡泡正在飞向中央大泡泡并与其融合，大泡泡颜色从蓝色柔和过渡到紫色，外环新增一段紫色轨迹。画面有液态融合感、光晕扩散和漂浮粒子。

### 图 3：Bubble Calendar 日历页

深色玻璃拟态日历页面，不使用传统数字格子，而是由一组圆形小泡泡组成。每颗泡泡代表一天，颜色代表当天主要时间类型。今天的泡泡带有发光光环。页面干净、梦幻、克制。

### 图 4：Daily Detail 当天详情页

某一天的时间总结页面。顶部显示日期，下方是柔和玻璃质感的统计区域，包括时间占比环形图、完整时间流 Timeline、不同时间类型的累计时长。风格保持深色液态玻璃，但比首页更适合阅读数据。

### 图 5：Bubble Explosion 今日结束

晚上氛围，中央大泡泡正在柔和爆裂，碎成许多发光小泡泡飞向四周，背景逐渐变暗，有月亮或夜晚光晕。画面表达“一天结束了”的仪式感，不要做成激烈爆炸。

## 23. 可直接复制给画图 AI 的总提示词

```text
设计一款名为 Time Bubble（时间泡泡）的手机 App。它不是 Todo、番茄钟或传统时间统计工具，而是一款记录“此刻我的时间流向哪里”的时间可视化应用。

核心画面是首页：深色梦幻渐变背景，带有漂浮粒子和柔和光晕。屏幕中央是一颗巨大的半透明玻璃泡泡，像液态玻璃和肥皂泡，具有 Apple Vision Pro / iOS Liquid Glass 的高级质感。泡泡中央显示当前时间类型的图标、名称和当前持续时间，例如“📖 学习 00:26:31”。这个时间表示当前这一段学习持续了多久，不是今日累计。

大泡泡外侧有一圈彩色液态时间环，表示今天时间流动的轨迹。它不是饼图，不是分类统计，而是按时间顺序排列的 Segment 轨迹，例如灰色无所事事、蓝色学习、紫色娱乐、蓝色学习、绿色工作。相同颜色可以重复出现，不要合并。

大泡泡周围漂浮多个小泡泡，分别代表学习、娱乐、工作、阅读、运动等时间类型。小泡泡只显示图标和名称，不显示累计时间。顶部只有日历入口和设置入口，不要底部导航，不要任务列表，不要大面积卡片，不要复杂文字。

整体视觉关键词：Glassmorphism、Dream UI、Liquid Glass、floating bubbles、soft glow、fluid motion、breathing, calm, magical, minimal, emotional, dark gradient background。

整体感觉应该像一个有生命的时间泡泡世界，而不是后台管理系统或效率工具。
```

## 24. 最重要的产品边界

请始终记住：

> 时间泡泡的核心不是“时间统计”，而是“时间流动”。

首页只回答：

> 现在，我的时间属于谁？

详情页才回答：

> 今天，我的时间最终流向了哪里？
